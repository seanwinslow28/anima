# Role
You are a research analyst evaluating image editing models for an animation production pipeline.

# Project Context
I produce 2D pencil-test animation frames. After generating a keyframe, I often need to make targeted edits — e.g., "change only the hand pose," "redraw the eyes with a different expression," "extend the paper canvas to add background," "fix the stylus angle without redrawing the face." Currently I either re-generate the whole frame (expensive, character drifts) or hand-fix in Procreate (slow). I want models that perform precise, instruction-driven edits while preserving the rest of the image exactly AND preserving the hand-drawn pencil-on-cream aesthetic.

# Known State as of 2026-05-24 (DO NOT re-survey — treat as given and EVALUATE / EXTEND)

The following findings come from independent 2026 comparisons (MimicPC, DigitalOcean, fal.ai, Oxen.ai's fine-tuning study, Wiro AI's 25-prompt test, Diffusion Doodles, ITPiran). Treat as the *current baseline*. Verify, evaluate for my use case, and find what they miss.

**Closed-source / API editing leaders:**
- **Google Nano Banana 2** — current production editor in my pipeline. ~$0.039/image. Better than GPT Image 2 for *scene-integrated* edits (lighting and environment cohesion) but loses to GPT Image 2 on face precision when locking a reference exactly. Resolution: 4K. Per Isa does AI's head-to-head test, NB2 wins category 3 (image editing) on integration quality.
- **OpenAI GPT Image 2** — wins on text rendering, multilingual scripts, prompt-adherence on complex briefs. Caps at 1K resolution and ~5× the cost of NB2 for standard tier. Better when goal is locking a face pixel-perfect from a reference.
- **ByteDance SeedEdit 3.0** — closed API, "faithful instruction following + content preservation" — independent reports say it's strong on instruction precision but I have no first-party data. Evaluate.

**Open-source editing leaders (best on a 24 GB consumer GPU):**
- **FLUX.1 Kontext [dev]** — 12B rectified flow transformer, open weights (non-commercial license — flag this for my portfolio context), iterative editing, best-in-class character preservation across scenes per BFL claims and independent confirmation. Strong on detailed prompts.
- **Qwen-Image-Edit (and Qwen-Image-Edit-Plus if released)** — 20B, Apache 2.0 (the critical license advantage), dual Qwen2.5-VL semantic + VAE appearance architecture. Best for precise targeted edits (color, style adjustments) without bleeding to unedited regions. Can also generate consistent character datasets for downstream LoRA training.
- **OmniGen2** — unified gen+edit, SOTA among open-source for complex instruction-based edits.
- **ByteDance Lance (2026-05-21, Apache 2.0, 3B)** — newly released. Native unified architecture; editing is one of three modalities (with understanding + generation). Evaluate explicitly for editing quality vs FLUX Kontext and Qwen-Image-Edit. This is the freshest open contender.

**Dimensional-win patterns confirmed by independent 2026 comparisons:**
- Color/style targeted tweaks: Qwen-Image-Edit cleanest
- Detailed-prompt completeness: FLUX Kontext edges out
- Style transfer with original preservation: Qwen + Nano Banana lead
- Open-source instruction precision: OmniGen2 best-in-class

**Critical gap none of the sources address (please solve):**
NONE of the 2026 head-to-head comparisons I've found explicitly tested *hand-drawn / pencil / non-photoreal preservation during edits*. Most testing is on photoreal subjects. This is the single most important question for my use case.

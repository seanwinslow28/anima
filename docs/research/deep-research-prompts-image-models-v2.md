## Prompt 2 v2 — Identity Preservation Deep-Dive

```
# Role
You are an AI research analyst specializing in identity and character preservation techniques for diffusion image models for 2D animation production.

# Project Context
I am conditioning image models to consistently render the same character ("Sean") across hundreds of frames in a 2D pencil-test animation. I have:
- 1 hero reference image of the character (3/4 head, full body, pencil-drawn on cream paper)
- ~30 approved keyframes already generated by Nano Banana 2 (could become a LoRA training set or extra references)
- A consumer rig with a 24 GB GPU (RTX 4090 equivalent)
- Goal: drop API cost by running in-betweens locally without losing character likeness OR the hand-drawn pencil aesthetic

# Known State as of 2026-05-24 (DO NOT re-survey — treat as given and EVALUATE / EXTEND)

The following findings come from a fresh practitioner pulse (Reddit r/StableDiffusion + YouTube tutorials Feb–May 2026 + recent ComfyUI/HF documentation). Treat these as the *current baseline*. Your job is to verify, evaluate against my specific use case, and find what they miss.

1. **Z-Image Turbo (Tencent)** — new architecture that has dropped LoRA character training requirements to 10–15 high-quality images (down from 20–50). Active ComfyUI tooling: comfyUI-Realtime-Lora supports it alongside FLUX/SDXL/WAN. Promptus AI YouTube tutorial "Character Consistency Solved Locally" (May 2026) and Next Diffusion + Apatero written walkthroughs exist. Evaluate explicitly: viability on a 24 GB GPU, quality vs FLUX LoRA, and aesthetic preservation for hand-drawn pencil style (not photoreal).

2. **Qwen-Image-Edit as a LoRA dataset generator** — emergent 2026 pattern: instead of training a LoRA from raw photos, use Qwen-Image-Edit (Apache 2.0, 20B, dual Qwen2.5-VL semantic + VAE appearance) to generate consistent character variants first, then train a LoRA on that clean synthetic dataset. A multi-angle LoRA trained on 96 viewpoints already exists publicly. Evaluate: how well this approach preserves a hand-drawn aesthetic, and whether the synthetic-dataset pipeline degrades character identity vs training on real keyframes.

3. **The 2026 production stack for film-grade character locking**:
   - Low-strength LoRA (0.6) for general body/hair/vibe
   - PuLID adapter (0.8) for precise facial features (no training required)
   - ControlNet (OpenPose) for posture control
   Known caveat: PuLID "can look pasted on if lighting doesn't match" — high risk for my unified pencil/cream aesthetic. Evaluate whether PuLID-FLUX, PuLID-Lightning, or any 2026 PuLID successor mitigates this for non-photoreal styles.

4. **ByteDance Lance (2026-05-21, Apache 2.0, 3B active params)** — newly released native unified multimodal model. Handles image+video understanding + generation + editing in one model. Trained on a 128×A100 budget. Evaluate explicitly for the in-between use case: does it support multi-image conditioning (start frame + end frame + style ref) natively? How does it perform on non-photoreal styles vs FLUX-Kontext?

5. **Models the v1 prompt asked about but appear to have stalled or been superseded in 2026 chatter** — please confirm whether these are still worth my time or skip:
   - InstantID, PhotoMaker v1/v2 (was the v1 ask still relevant for FLUX/SD3.5/Z-Image?)
   - ConsiStory, StoryDiffusion, IC-LoRA (any meaningful 2026 activity?)
   - HiDream-I1, OmniGen original, DeepSeek Janus-Pro, Lumina-Image, PixArt-Sigma, Sana (NVIDIA) — flag any that have a 2026 successor I should look at instead

# Research Task
Produce a 2025–2026 guide to identity preservation for character-consistent image generation in a hand-drawn animation pipeline, covering:

## 1. Adapter-based methods (no training required)
IP-Adapter family, InstantID, PuLID family (with explicit focus on whether 2026 successors fix the "pasted-on" lighting issue for non-photoreal styles), PhotoMaker v2, ConsiStory/StoryDiffusion/IC-LoRA — for each: how it works, supported base models (must include FLUX, Qwen-Image, Z-Image), VRAM, similarity preservation 1–5, pose/expression edit-ability while preserving identity, hand-drawn-style preservation 1–5, recent independent comparisons. Flag any 2026 entrant not in this list.

## 2. Training-based methods
LoRA character training across FLUX, Qwen-Image, Z-Image, SDXL, and SD3.5: optimal dataset size in 2026 (cite the new 10–15 minimum where it applies), augmentation strategy, captioning approach, training time on a 4090, recommended trainers (kohya_ss, ai-toolkit, OneTrainer, SimpleTuner, comfyUI-Realtime-Lora). Specifically address: does training on synthetic Qwen-Image-Edit-generated data match training on real keyframes for identity preservation?

## 3. Hybrid / combined approaches
The LoRA + PuLID + ControlNet stack — independent verification of its reported quality. Alternative stacks for the in-between use case (start frame + end frame + reference): multi-reference attention manipulation, latent blending, differential diffusion. Style LoRA + character LoRA stacking specifically for hand-drawn pencil aesthetic.

## 4. In-between-specific 2026 workflows
Given start frame + end frame + character reference, what's the state of the art for generating a frame that (a) looks like the same character, (b) smoothly interpolates pose/expression, (c) preserves a specific art style (pencil / hand-drawn)? Evaluate ToonCrafter, AnimeInbet, EISAI, FILM, RIFE-derived models, custom diffusion-based interpolation, AND the new unified models (Lance, OmniGen2) for this specific job. Cite ComfyUI workflows where they exist.

## 5. Three recipe recommendations for my rig (24 GB)
- **Fast & flexible** — adapter-only, no training, evaluate PuLID-FLUX vs the Lance unified approach
- **High-fidelity** — LoRA-trained on existing 30 keyframes + adapter (compare Z-Image Turbo LoRA vs FLUX LoRA vs SD3.5 LoRA paths)
- **Style-locked** — character LoRA + style LoRA + control conditioning, optimized for hand-drawn pencil aesthetic preservation

For each: exact models/files, ComfyUI graph description, expected quality 1–5, common failure modes, cost per frame (electricity + amortized capex), known character-drift issues across long sequences.

# Output Format
- Executive summary (5 bullets) with top recommendation
- Comparison table: technique × base model × quality × setup difficulty × VRAM × hand-drawn-style score
- Detailed sections
- Three pipeline recipes with full ComfyUI graph descriptions
- "Verdict on the known state" section: confirm/correct each of the 5 items in the Known State block
- Citations for every claim with URLs and dates (prefer post-2026-01 sources; flag earlier sources)

# Hard Rules
- Prioritize 2026 sources; treat anything pre-2025 as "may be superseded" unless still actively maintained
- Cite HF model cards, GitHub repos, arXiv papers, independent benchmarks, ComfyUI workflow repos
- Verify availability — flag if a method is "research only" or has been abandoned
- Don't conflate similar-named methods (InstantID ≠ InstantStyle, IP-Adapter-FaceID ≠ IP-Adapter-Plus)
- For each model named in the Known State block, return a verdict: "confirmed leading," "confirmed but caveats," "superseded by X," or "evidence too thin"
- Pencil/hand-drawn aesthetic preservation is non-negotiable — penalize models that only have photoreal evidence

## Prompt 3 v2 — Image Editing Deep-Dive

```
# Role
You are a research analyst evaluating image editing models for an animation production pipeline.

# Project Context
I produce 2D pencil-test animation frames. After generating a keyframe, I often need to make targeted edits — e.g., "change only the hand pose," "redraw the eyes with a different expression," "extend the paper canvas to add background," "fix the stylus angle without redrawing the face." Currently I either re-generate the whole frame (expensive, character drifts) or hand-fix in Procreate (slow). I want models that perform precise, instruction-driven edits while preserving the rest of the image exactly AND preserving the hand-drawn pencil-on-cream aesthetic.

# Known State as of 2026-05-24 (DO NOT re-survey — treat as given and EVALUATE / EXTEND)

The following findings come from independent 2026 comparisons (MimicPC, DigitalOcean, fal.ai, Oxen.ai's fine-tuning study, Wiro AI's 25-prompt test, Diffusion Doodles, ITPiran). Treat as the *current baseline*. Verify, evaluate for my use case, and find what they miss.

**Closed-source / API editing leaders:**
- **Google Nano Banana 2** — current production editor in my pipeline. ~$0.039/image. Better than GPT Image 2 for *scene-integrated* edits (lighting and environment cohesion) but loses to GPT Image 2 on face precision when locking a reference exactly. Resolution: 4K. Per Isa does AI's head-to-head test, NB2 wins category 3 (image editing) on integration quality.
- **OpenAI GPT Image 2** — wins on text rendering, multilingual scripts, prompt-adherence on complex briefs. Caps at 1K resolution and ~5× the cost of NB2 for standard tier. Better when goal is locking a face pixel-perfect from a reference.
- **ByteDance SeedEdit 3.0** — closed API, "faithful instruction following + content preservation" — independent reports say it's strong on instruction precision but I have no first-party data. Evaluate.

**Open-source editing leaders (best on a 24 GB consumer GPU):**
- **FLUX.1 Kontext [dev]** — 12B rectified flow transformer, open weights (non-commercial license — flag this for my portfolio context), iterative editing, best-in-class character preservation across scenes per BFL claims and independent confirmation. Strong on detailed prompts.
- **Qwen-Image-Edit (and Qwen-Image-Edit-Plus if released)** — 20B, Apache 2.0 (the critical license advantage), dual Qwen2.5-VL semantic + VAE appearance architecture. Best for precise targeted edits (color, style adjustments) without bleeding to unedited regions. Can also generate consistent character datasets for downstream LoRA training.
- **OmniGen2** — unified gen+edit, SOTA among open-source for complex instruction-based edits.
- **ByteDance Lance (2026-05-21, Apache 2.0, 3B)** — newly released. Native unified architecture; editing is one of three modalities (with understanding + generation). Evaluate explicitly for editing quality vs FLUX Kontext and Qwen-Image-Edit. This is the freshest open contender.

**Dimensional-win patterns confirmed by independent 2026 comparisons:**
- Color/style targeted tweaks: Qwen-Image-Edit cleanest
- Detailed-prompt completeness: FLUX Kontext edges out
- Style transfer with original preservation: Qwen + Nano Banana lead
- Open-source instruction precision: OmniGen2 best-in-class

**Critical gap none of the sources address (please solve):**
NONE of the 2026 head-to-head comparisons I've found explicitly tested *hand-drawn / pencil / non-photoreal preservation during edits*. Most testing is on photoreal subjects. This is the single most important question for my use case.

# Research Task
Produce a 2025–2026 state-of-the-art report on instruction-driven and region-based image editing, with explicit emphasis on hand-drawn / non-photorealistic preservation.

## 1. Verify and extend the closed-source landscape
Confirm or correct the Known State for Nano Banana 2, GPT Image 2, SeedEdit 3.0. Add any 2026 entrant not listed (Recraft, Ideogram Magic Fill, Adobe Firefly Image 4 / Generative Fill API, others). For each: edit invocation (mask vs instruction vs reference), preservation of un-edited areas, identity preservation, multi-edit chains, cost per edit, API stability, and — critically — independent reports of behavior on non-photoreal/hand-drawn input.

## 2. Verify and extend the open-source landscape
For FLUX Kontext, Qwen-Image-Edit, OmniGen2, Lance, plus SD3.5 + ControlNet-Inpaint, BrushNet, PowerPaint, HiDream-Edit (if released), InstructPix2Pix, MagicBrush, UltraEdit, Anole, Emu-Edit (if open-released). Same evaluation axes as section 1, plus: install/run reqs on 24 GB, license details (commercial OK or research-only — Apache 2.0 strongly preferred for my portfolio use), active ecosystem health.

## 3. The hand-drawn / pencil preservation question (highest priority)
This is the gap in current benchmarks. Investigate:
- Which models in sections 1+2 have documented evidence (independent or vendor) of preserving a hand-drawn / pencil / illustrative aesthetic through an instruction edit?
- Which models "slick up" non-photoreal inputs (introducing digital sheen, smoothing pencil texture)?
- Are there practitioner reports from indie illustrators or animators (Reddit, YouTube, Hugging Face Discord, Civitai) specifically about hand-drawn edits?
- Are there style-locked editing techniques (style LoRA stacked with the editing model) that solve this?

## 4. Editing techniques (model-agnostic) for 2026
- Inpainting with masks — best 2026 pipelines
- Instruction-edit vs reference-edit vs mask-edit — when to use which
- Latent blending and seed-locking for preservation
- Differential diffusion / region-aware sampling
- Multi-region simultaneous edits

## 5. Three recommended editing setups for my use case
- **Cheapest reliable closed-source** edit per frame — assume NB2 is the baseline; only recommend an alternative if it materially beats NB2 on hand-drawn preservation at lower cost
- **Best-quality open-source** on a 24 GB GPU — explicit comparison: FLUX Kontext vs Qwen-Image-Edit vs OmniGen2 vs Lance for the pencil-test edit use case
- **Hybrid pipeline** — open-source bulk edits + closed-source for hero shots; specify which open model owns which edit type

# Output Format
- Executive summary
- "Verdict on the known state" section: confirm/correct/supersede each item from the Known State block
- Comparison table: model × edit type × instruction-follow rating × preservation rating × cost/VRAM × hand-drawn aesthetic score (THIS COLUMN IS NON-NEGOTIABLE)
- Detailed model sections
- Three recommended setups
- Citations for every claim with URLs and dates

# Hard Rules
- Cite all sources with URLs and dates; prefer post-2026-01 sources
- Link to actual edit-result examples when sources provide them
- Note explicitly whether each model has been tested on non-photorealistic styles
- Distinguish models that edit by mask vs by instruction vs by reference image
- License clarity required: Apache 2.0 / MIT / non-commercial / API-only — flag each
- For each Known State entry, return a verdict: "confirmed leading," "confirmed but caveats," "superseded by X," or "evidence too thin"

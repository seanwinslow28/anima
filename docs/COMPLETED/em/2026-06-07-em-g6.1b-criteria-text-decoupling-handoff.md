## Part A — the decoupling (non-costed code change)

### A1. `pipeline/agents/vision_critic.py`

1. Add a method mirroring `_attach_references` (lines 220–229):

   ```python
   def _attach_criteria_text(self, t2_cfg: dict) -> bool:
       """Whether to attach the IR/AC criteria block (the text handles Em cites),
       INDEPENDENT of reference images. Default False. Decoupled from
       attach_references (G6.1b, 2026-06-07): the 2026-06-07 re-baseline proved
       the criteria block was inert reference-blind because it shared the
       attach_references gate. Criteria-text grounds Em's citations without the
       reference-image precision regression (2026-06-04 references re-test)."""
       return bool(t2_cfg.get("attach_criteria_text", False))
   ```

2. Change the gate at line 381 so criteria attaches when **either** flag is set (preserves the existing references-on behavior, which attached both, and adds independent criteria-text):

   ```python
   if self._attach_criteria_text(t2_cfg) or self._attach_references(t2_cfg):
       criteria_block = self._criteria_block(ctx)
       if criteria_block:
           sections.append(criteria_block)
   ```

3. **Do not touch** `run()` lines 129–130 — reference *images* stay gated on `_attach_references` only. This is the decoupling guarantee: `attach_criteria_text=True, attach_references=False` ⇒ criteria text present, **zero images attached** (`n_references == 0`, the "Reference plates" section at line 363 stays off).

4. **Small required wording fix in `_criteria_block`** (line ~295): the text *"a rule the references confirm is honored is not a citation"* assumes reference images are attached. In criteria-text-only mode there are none. Reword to *"a rule the frame honors is not a citation; cite only what you actually see drift on."* Keep everything else (the "copy the handle verbatim / do NOT substitute HF-SF / do NOT invent a handle" instruction at lines 290–299 is the whole point — leave it intact).

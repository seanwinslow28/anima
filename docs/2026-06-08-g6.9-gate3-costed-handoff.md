## What this run measures

Em's **verdicts** (0.97/1.00/0.00) and **citations** (0.97) are measured. Her **constructive output** — the prompt diffs she proposes — is not, *empirically*. Gate 3 closes that: take Em's proposed corrective clause, **apply it → re-generate the frame → re-critique**, and ask per re-roll: **did the defect clear AND did identity/style hold?** N re-rolls → a clear-rate **band** (generation is stochastic). Headline = **fix-rate**, read as normalized lift (below).

The machinery is built and stub-green (`evals/vision_critic/patch_efficacy.py`, `prompt_apply.py`); this run turns it live.

## Decisions already locked (do not re-litigate)

| # | Decision | Value |
|---|----------|-------|
| Attribution | Floor-corrected | **Null/placebo arm** on the first run. Headline = **normalized lift `(em − null)/(golden − null)`**, raw em/golden/null clear-rate bands **per class**. |
| First run | Sample + N | **`--arm both+null --sample 12 --rerolls 3`** (2 cases/class × 6 classes; the affordable first read). |
| Apply mechanism | Clean-pair splice | Deterministic; robust to the corpus's 3 defect grammars. The **null arm measures its floor**; the golden control bounds it; identity-held keeps it honest. View-correctness is label-side (no-regen). |

**Why the null arm (the core of the attribution lock).** The clean-pair base can regenerate a clean frame even from a *useless* clause (the scene itself isn't defective), inflating raw fix-rate. The null arm splices a fixed defect-orthogonal placebo (`_NULL_CLAUSE = "with a calm, evenly-lit, neutral background"`) through the same mechanism, measuring the floor. `lift = (em − null)/(golden − null)`: **1.0** = em matches the golden's lift over the floor, **0.0** = em no better than placebo, **<0** = worse. Per class, **`golden ≈ null` ⇒ no discriminative power there** — a measured finding (`discriminative: false`), not a guess.

# Em — Baseline Trace (2026-05-26)

This is the first-run trace of Em — anima's T2 vision critic — against the Act 1 reference run and one Act 2 hero anchor. It exists because the implementation prompt's "Verify" section for commit 8 made the baseline a hard requirement: *Baseline must exist before Em ships.*

The shape of the responses, the routing logic, and the patch-staging behavior are what this trace captures. The model-quality comparison — Gemini 3.1 Pro vs Claude Sonnet vs Opus 4.7 on Sean's specific corpus — is commit 8b's job, and that bake-off will diff against this trace.

## Run environment, 2026-05-26

Em shipped on a machine without the Anti-Gravity CLI installed (`which anti-gravity` returned nothing) and without `claude-agent-sdk` or `anthropic` Python packages in the venv (`pip list` confirmed). `ANTHROPIC_API_KEY` was unset. Both runner modules — `pipeline/agents/cli_runners.py` for Gemini, `pipeline/agents/sdk_runners.py` for Opus — fell back to their deterministic stubs, which are designed for exactly this case: keep tests green, exercise the full routing logic, write the right shape into the sidecar JSON, stage patches that look real to the CLI viewer. The model behind the words isn't doing work yet; the contract layer around it is doing all of it.

Sean's plan is to install the CLIs and the SDK as part of commit 8b's bake-off prep. This trace captures the contract baseline; the model baseline comes next.

## Run 1 — Act 1 F06, no impact tags

**Fixture:** `tests/fixtures/manifest_em_act1.yaml`
**Frame:** `runs/run_2026-04-04_174805/approved/PT_A1_F06_key.png` — the shipped Act 1 F06 keyframe (Sean looking down at the stylus, contemplative, head tilted ~15°).
**Beat description:** "F06 — glance down, eyes on stylus, contemplative; head tilts ~15 degrees downward"
**Impact tags:** `[]`
**Confidence threshold:** 0.7

Em ran. Gemini stub returned a borderline verdict at confidence 0.65 (the deterministic Gemini-stub baseline). 0.65 < 0.7, so the escalation hatch fired. Opus stub returned a borderline verdict at confidence 0.78 (the deterministic Opus-stub baseline). The Opus result wins.

The sidecar JSON Em wrote into `runs/run_em_verify_act1/.cache/`:

```json
{
  "outputs": {
    "verdict": "borderline",
    "reasoning": "STUB FALLBACK — claude-agent-sdk + anthropic SDK both unavailable. ...",
    "confidence": 0.78
  },
  "tier": "draft",
  "cites_criteria": ["AC01"],
  "notes": "em@phase_5_generate frame=F06 (escalated)"
}
```

The `(escalated)` notes marker is the load-bearing observable — it tells the museum walkthrough later which path fired. Both runs (first and second) completed in ~50ms each. The second invocation served from the cache; the cache key (per commit 4's `cache_key()` function) folds in tier, criteria hash, and normalized inputs.

No patches staged (the stub returns an empty `proposed_patches` list), so no `manifest.lock.yaml` was written. That's the correct behavior — patches don't get written for pass-through critiques.

## Run 2 — Patch-staging end-to-end with a mocked Gemini

**Fixture:** ad-hoc verification script (`runs/run_em_patches_verify/`)
**Frame:** Same F06 keyframe.
**Mocked Gemini:** Returns a borderline verdict at confidence 0.91 (above threshold, so no escalation) with one proposed patch tightening the F06 prompt against the SF01 style-drift mode.

The `pipeline/agents/patch_stager.py` post-run hook fired and wrote the patch into `runs/run_em_patches_verify/manifest.lock.yaml`:

```yaml
proposed_patches:
- target: manifest.lock.yaml
  path: act1.keyframes[1].pose
  operation: set
  value: F06 glance down with varied line weight 1-3px, visible construction
    lines under cheekbone, cross-hatching on jaw shadow
  rationale: 'Em: line weight + construction lines drifted from anchor;
    tightening prompt to re-anchor style.'
  proposed_by: em-vision-critic
  cites_criteria:
  - AC01
  node_id: em_f06_critic
```

The `python -m pipeline.cli patches list --run-dir runs/run_em_patches_verify` command rendered the patch grouped by persona with clean indentation. Sean reviews this list before any manifest mutation lands — the auto-apply lock from v2 brainstorm §2.5 is structural.

## Failure modes the stub fallback cannot test

The stub responses are deterministic borderline verdicts at fixed confidences. They cannot tell us:

- Whether Gemini 3.1 Pro actually produces useful prompt diffs against pencil-test aesthetic drift.
- Whether Opus 4.7 disagrees with Gemini on identity drift in any meaningful way.
- Whether either model can cite acceptance criteria from `acceptance_criteria.json` accurately enough that the `cites_criteria` invariant pulls real weight.
- Latency under real load — every stub call here returned in ~0ms; real calls will sit at 5–60s.
- Whether the `default_context_files` block (PHILOSOPHY.md + CLAUDE.md + the architecture doc) actually shifts the critique away from generic recommendations.

All five resolve in commit 8b's bake-off, run against Sean's real Act 1 + Act 2 corpus with the CLI + SDK installed. The defect set comes from intentionally-red cases (style drift, identity drift, stylus continuity breaks) plus clean cases from the shipped Act 1 hero loop. Metrics: revision-minutes-to-fix, blind-preference scores, `proposed_patches:` accept-rate, `cites_criteria` citation density.

## What this baseline locks in

The trace's load-bearing claim is narrow: anima's T2 critic contract works end-to-end under stub-only conditions, on real run dirs, with the DAG runner's cache and hook system both engaged.
